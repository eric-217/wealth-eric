# 财富管理客户报表系统 · 一键部署指南

> 给同事看的。复制本文到 WorkBuddy 新对话，一条指令完成全部部署。

---

## 一、这是什么

```
客户打开网页 → 手机号+密码登录 → 看到专属财务报表
你更新数据 → git push → 30秒自动上线 → 客户刷新即看
```

**免费**：GitHub 私有仓库 + Cloudflare Pages 永久免费套餐。  
**零运维**：不需要服务器、不需要数据库、不需要域名。  
**专业感**：密码保护、多客户隔离、手机自适应。

## 二、让 WorkBuddy 帮你搭

打开 WorkBuddy 新对话，复制下面这句话发给它：

```
我需要搭建一个客户财富管理报表系统。

请按以下结构创建项目文件夹，并生成全部文件：

1. 文件夹：clients/template/，里面包含：
   - 菜单.html（登录后看到的导航页）
   - 家庭财务总览_手机版.html（总览仪表盘）
   - 固收现金流明细.html、保险明细.html、负债明细.html
   - 保单管理.html、配置建议.html、待办事项.html
   以上全部用 {{占位符}} 代替真实数据，保留完整页面结构。

2. 根目录创建：index.html（首页+登录入口）、login.html（手机号+密码登录）、dashboard.html（认证后加载客户页面）

3. assets/auth.js：客户认证系统，用 SHA-256 存储密码。预置一个模板账号（手机号13800000000，密码000000）。

4. 初始化 Git 仓库，推送到我的 GitHub 账号。
```

WorkBuddy 知道全部技术细节，会自动生成所有代码。

## 三、关联 Cloudflare Pages

1. 打开 [Cloudflare Dashboard](https://dash.cloudflare.com) → Workers & Pages → Create
2. 点底部小字 "Looking to deploy Pages? Get started"
3. 连接你的 GitHub 仓库
4. 构建设置：Framework Preset 选 **None**，Build command 和 Output directory 留空
5. 点 Deploy

完成。会得到一个 `xxx.pages.dev` 域名。

## 四、新增客户

在 `clients/` 下复制 template 文件夹，改名（如 `clients/clientA/`），编辑 HTML 把 `{{...}}` 换成真实数据。然后在 `assets/auth.js` 加一行：

```js
'13900000000': {
  name: '客户名称',
  path: 'clientA/家庭财务总览_手机版.html',
  hash: '这里填 SHA-256 哈希'
}
```

哈希用浏览器控制台生成：`crypto.subtle.digest('SHA-256', new TextEncoder().encode('密码')).then(h => console.log(Array.from(new Uint8Array(h)).map(b => b.toString(16).padStart(2,'0')).join('')))`

## 五、日常更新

在本地改了文件 → 告诉 WorkBuddy "推送" → `git push` → Cloudflare 30 秒自动部署。

## 六、已建好的角色和规则

系统内置了 8 个 AI 角色，会自动切换：

| 角色 | 做什么 |
|------|------|
| 会计 | 每笔交易等号两边同时变 |
| 财务总管 | 资产负债全貌 |
| 理财规划师 | 资金排程、财富自由 |
| 保险专家 | 保单分析、条款提取 |
| 港险配置 | 分红规则、计划书 |
| 税务顾问 | CRS、外汇、个税 |
| 法律提示 | 受益人、继承 |
| 客户顾问 | 面向客户的输出语气 |

## 七、核心规则（系统自动遵守）

- 用增值支付支出，不伤本金
- 资金变动必须两边同时更新
- 所有报告禁用英文缩写（现金价值≠CV）
- 每份保单必须分析附加险（绝不遗漏）
- 客户菜单：有什么展示什么，没有不显示
- 禁止展示完整身份证号、姓名、保单号
