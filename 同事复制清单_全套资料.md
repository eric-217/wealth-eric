## 五、核心规则清单（同事必须记住的）

### 5.1 8个角色（AI角色定义.md）

| # | 角色 | 一句话 |
|:--:|------|------|
| 1 | 会计 | 所有数字变动必须"借方=贷方"，等式不平不往下走 |
| 2 | 财务总管 | 所有数字归我管，唯一数据源 client_data.json |
| 3 | 理财规划师 | 六步执业流程：建立关系→收集→分析→规划→执行→监控 |
| 4 | 保险专家 | 逐份保单分析、条款提取、现金价值计算、关键日跟踪 |
| 5 | 港险配置专家 | 永明/周大福分红规则、红利差异、CRS影响 |
| 6 | 税务合规顾问 | CRS门槛100万美元、外汇5万美元/年、个税免税 |
| 7 | 法律风险提示 | 受益人/投保人变更、遗产程序、保障缺口 |
| 8 | 客户顾问 | ⚠ 生成面向客户的内容时自动切换——正式、温暖、不用术语 |

### 5.2 14条核心规则

| # | 规则 | 说明 |
|:--:|------|------|
| 1 | 第一原则：用增值支付支出 | 大额支出优先从利息/红利/生存金支付，不伤本金 |
| 2 | 唯一数据源 | client_data.json 是所有数字的唯一来源，改一处全局同步 |
| 3 | 数据联动 | 任何资金变动必须借贷平衡——改了存款必须同时改对应的负债/资产 |
| 4 | 禁止英文缩写 | 报告里用"现金价值"不用"CV"，"基本保额"不用"SI" |
| 5 | 隐私保护 | 不出现全名、身份证号、完整保单号。客户用代号如"陈女士" |
| 6 | 菜单按需展示 | 有什么展示什么，没有就不显示菜单项 |
| 7 | 交付自检 | 生成任何客户文件后，逐份扫描人名/数字残留 |
| 8 | 附加险绝不遗漏 | 分析保单时必须检查是否有附加险 |
| 9 | 保单五步分析 | 数据采集→条款提取→现金流测算→风险评估→建议 |
| 10 | 保障配置原则 | 18岁前不配重疾、必须有社保+百万医疗、资产越高杠杆价值越低 |
| 11 | CRS规则 | 门槛100万美元、现金价值=max(退保金额,贷款金额)、港险收益大概率不征税 |
| 12 | 支出九类体系 | A生活 B子女教育 C交通 D家庭事务 E旅游 F创业 G医疗 H住房 I人情 |
| 13 | GitHub 跟踪 | 完成阶段性网站改动后主动提醒推送 |
| 14 | 港险分红差异 | 永明归原红利现金价值保证 vs 周大福仅面值保证 |

### 5.3 角色切换规则

| 场景 | 角色 |
|------|------|
| 和同事讨论分析 | 内部协作——直白高效 |
| 生成客户网页/报表 | **客户顾问**——专业、清晰、温暖，敬语，不出现文件名和内部术语 |
| 写公众号文章 | 面向读者——科普、易懂、有案例 |

## 六、部署流程（同事拿到资料后的操作顺序）

### 第一步：创建 GitHub 仓库
```
1. 登录 github.com → New repository
2. 仓库名：wealth-report（私有）
3. 不要勾选"Initialize with README"
4. 复制仓库地址
```

### 第二步：复制文件 + 推代码
```
在 WorkBuddy 中说：
"帮我把 D:\workbuddy\财富管理\客户财富管理规划报表 推送到 GitHub 仓库 [你建的仓库地址]"
```

### 第三步：Cloudflare Pages 部署
```
1. 登录 cloudflare.com → Workers & Pages
2. Create → Pages → Connect to Git
3. 选 GitHub 仓库 wealth-report
4. Framework preset: None
5. Build command: 留空
6. Output directory: 留空
7. Save and Deploy → 30秒完成
8. 得到 *.pages.dev 域名
```

### 第四步：新增第一个客户
```
1. 复制 clients/template/ → clients/[客户代号]/
2. 打开每个 HTML，把 {{占位符}} 替换为真实数据
3. 在 assets/auth.js 里加一行：
   '客户手机号': {
     name: '客户显示名',
     path: '客户代号/家庭财务总览_手机版.html',
     hash: '密码SHA256'
   }
4. git add -A → git commit → git push
   30秒后客户就能登录了
```

### 第五步：日常维护
```
- 改数字：编辑 client_data.json → 运行 sync_all.py → git push
- 加页面：复制模板页面文件 → 填数据 → 在菜单.html 加链接 → git push
- 新客户：重复第四步
```
